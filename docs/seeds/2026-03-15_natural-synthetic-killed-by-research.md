## 소재 요약

"첨가물이 천연인지 합성인지 보여주자"는 기능 요청 → 3단계 규제 리서치 → 4자(Claude·Gemini·Gemini Code Assist·ChatGPT) 합의 감리 → **코드 한 줄 안 쓰고 만장일치 기각**.

### 발견 과정
1. DB에 B1(화학적합성)/B2(천연) 코드가 이미 있어서 "쉽겠다" → 구현 직전 리서치 시작
2. 식약처 [별표 4]는 분류 목록이 아니라 절차 규정이었음
3. 한국은 2018년에 천연/합성 이진 라벨링을 **폐지**했음 (케모포비아 악용 방지)
4. B1/B2는 등록 시점 행정 분류일 뿐, 제품 단위 제조 기원을 보장하지 않음 (비타민C가 B1이지만 천연 추출 가능)
5. EU/Codex는 출처 구분 유지하나, 한국 DB로는 매핑 불가

### ChatGPT의 킬러 한 줄
> "첵잇이 보여줄 수 있는 것은 '분류 코드'인데, 사용자가 받아들이는 것은 거의 항상 '안전성 신호'이다."

## story 관점 (운영/브랜드)

**제목안**: "그 기능, 만들지 않기로 했습니다"

- 도입: 소비자 80%가 "천연 첨가물이 더 안전하다"고 믿는 현실
- 전개: 우리도 DB에 코드가 있으니 당연히 보여주려 했다
- 반전: 식약처가 2018년에 이 분류를 폐지한 이유가 바로 그 "믿음" 때문
- 결론: 정보 제공이 아니라 오해 유발이 될 수 있는 기능은 만들지 않는 것도 서비스의 책임
- 메시지: "할 수 있다"와 "해야 한다"는 다르다. 첵잇은 후자만 한다.

## tech 관점 (기술/프로세스)

**제목안**: "4개 AI가 만장일치로 기각한 기능의 해부"

- **리서치 비용 vs 구현 후 롤백 비용**: 반나절 리서치로 수주일 구현+법적리스크 회피
- **4자 감리 프로세스의 실전 검증**: Claude(구현자) → Gemini(감리) → Gemini Code Assist → ChatGPT, 각자 고유 기여
  - Claude: 3단계 규제 역사 발굴
  - Gemini: "정보 소유권을 식약처에 두라" 표현 세척
  - Gemini Code Assist: "용도 번역이 이미 클린라벨" 대안 제시
  - ChatGPT: "의미론 함정" — 코드를 보여줘도 안전성 신호로 읽힌다
- **DB-level 검증 기법**: KeyTextTransform + Trim + Replace로 28만건 JSON 필드 교차검증 (같은 세션에서 수행)
- **invisible character 디버깅**: API 데이터 내 tab/newline 발견 → hex dump로 추적
