# 같은 물질, 다른 UUID — CAS 번호로 동의어를 잡아내다

## 소재 요약

식품첨가물 DB에서 "같은 화학물질인데 UUID가 다른" 문제를 해결하기 위해 SubstanceMaster 설계에 착수.
초기 추정치 71건이 정밀 조사에서 88건으로 수정되고, CAS 전수 조사로 name_ko만으로는 발견 불가능한 동의어 4쌍(β-카로틴/카로틴, 결정셀룰로스/분말셀룰로스 등)을 포착.
에탄올제제 FK 연결에서는 "DB에 이미 있는 데이터를 새로 만들 뻔한" 사례도 발생.

## story (운영 블로그) 관점

- **"구연산이 5개?"** — 같은 물질이 분류 체계(B코드/E코드)마다 별도 등재되는 식약처 데이터 구조의 현실
- 에탄올제제: 접미사 하나("제제") 때문에 자동 매칭이 실패하고, 이미 있는 데이터를 새로 만들려고 했던 순간
- "71개라고 했는데 88개였습니다" — 초기 추정치를 그냥 쓰지 않고 재검증한 이유
- CAS(Chemical Abstracts Service) 번호라는 국제 식별자가 이름만으로 놓치는 동의어를 잡아내는 이야기

## tech (기술 블로그) 관점

- UUID 단편화 문제와 마스터 테이블 설계 패턴 (IC → SubstanceMaster ManyToOne)
- CAS 기반 그루핑 vs name_ko 기반 그루핑: 커버리지 72%(63/88) vs 100%이지만 동의어 누락 위험
- 그루핑 데이터 준비 전략: 실제 서비스 영향 범위(AUS FK 연결)에만 집중하여 88건으로 스코핑
- CAS 미보유 25건의 "알려진 한계" — 구조적으로 발견 불가능한 동의어를 문서로 남기는 방법
- 설계만 확정하고 마이그레이션을 미루는 판단 (frontend-pivot 이후 스케줄링)
