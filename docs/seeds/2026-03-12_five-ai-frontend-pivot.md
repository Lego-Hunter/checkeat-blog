## 소재 요약

창업자가 직접 "내 서비스인데 신뢰가 안 간다"고 판단 → 5개 AI(Claude/ChatGPT/Gemini/Gemini Code Assist/Claude Code)에게 동일 브리핑 → 독립 의견 수렴 → 8개 만장일치 + 5개 갈림점 도출 → 문서 기반 중재 → 3자 감리(Claude/ChatGPT/Gemini) → 실행계획 확정 → 당일 구현 완료.

## story (운영 관점)

- **"문서의 약속 vs 화면의 현실"** — vision.md에서 "배려형 서비스"를 약속했지만 실제 화면은 데이터 덤프
- 창업자가 자기 서비스에 대해 냉정한 진단을 내린 순간
- 5개 AI 의견 종합 과정 — 어떤 질문을 했고, 어떤 패턴이 나왔는지
- "유도가 아니라 채움" — 시드 카테고리 전략에서 사용자가 AI 의견을 교정한 사례

## tech (기술 관점)

- 5AI 동시 투입 방법론 — 같은 브리핑, 독립 응답, 교차 검증
- 갈림점 해소 기준: "문서의 약속에 충실한 방향" (strategy-identity.md 기반 중재)
- 3자 감리 프로세스 — 감리 결과를 "반영/보류/기각"으로 분류하는 구조
- FP-1 구현 상세: extractAllergens 기본 노출, 역방향 프로필 넛지, 데이터 부재 처리
- FP-2: %DV 영양 게이지, 서버사이드 알레르겐 필터, 3단계 정보 위계
- FP-3: 홈 리부트 (시나리오 카드 3개 + 예시 검색 칩 + About 푸터 토글), 온보딩 4단계→3단계 공감형 재설계 (페르소나 기반 분기)
- 마이크로카피 톤 설계 — "차분한 동행자" vs "같이 장보는 친구" (감리 피드백으로 결정)
- 기술 Polish: React.memo(HighlightedText/RiskTagBadge) + LEGAL_ALLERGENS E2E 정합성 테스트
