## Phase 3: 매칭 — 700만 번의 대조

| # | 제목 | 시리즈 | 핵심 갈림길 | 대응 story/ |
|---|------|--------|------------|-------------|
| P3-1 | 결정성 매칭 엔진 설계: 퍼지 매칭을 버린 이유 | public-data | 유사도 기반 vs 단계별 결정성(exact→contains→bracket→alias) | 10화, 11화 |
| P3-2 | 2-Pass Alias 정제: 데이터가 데이터를 하이재킹할 때 | public-data | I2520 사전의 alias 충돌 → 1회 루프로 안 되는 이유 | 10화 |
